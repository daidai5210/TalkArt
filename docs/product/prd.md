## 2. 背景与目标

### 2.1 背景

市场上不存在「纯语音控制 + AI Agent Function Calling + SVG 矢量绘图」的 Web 产品。传统 SVG 编辑器依赖鼠标键盘，学习曲线陡峭；AI 文生图工具输出不可编辑的栅格图。TalkArt 填补这个空白。

详见上游研究：`01-market-research.md`、`02-competitor-analysis.md`。

### 2.2 目标用户

所有人——不想学复杂绘图软件的大众创作者。自然覆盖无障碍场景。

详见上游研究：`03-target-users-and-personas.md`。

### 2.3 核心痛点

- 绘图软件学习曲线陡峭
- "我知道想画什么但画不出来"
- 无障碍用户被排除在绘图创作之外
- AI 文生图不可精确控制

详见上游研究：`04-pain-points-and-opportunities.md`。

### 2.4 一句话目标

> 说话就能画矢量图——AI Agent 理解你的创作意图，精确执行 SVG 绘图操作。

## 3. 产品概述

### 3.1 核心交互流程

```
唤醒词("小智小智")
  ↓
语音输入 → Web Speech API 转文字
  ↓
AI Agent (LLM) 分析意图
  ↓
多轮反问确认需求
  ├─ "你是想画一个红色圆形，放在画布中间，对吗？"
  ├─ 用户: "对，大一点"
  └─ "好的，一个大的红色圆形在中间。开始吧？"
  ↓
用户说"开始吧"/"可以了"
  ↓
LLM Function Calling → 选择工具 + 填充参数
  ↓
工具函数执行 → SVG 渲染到画布
```

### 3.2 关键设计决策

| 决策 | 内容 | 原因 |
|---|---|---|
| 唤醒词 | "小智"/"小zhi" 同音匹配 | 用户指定，多元识别 |
| 确认机制 | AI Agent 多轮反问 | 防止画错，零键盘鼠标无法手动纠错 |
| 执行触发 | "开始吧"/"可以了" | 用户明确确认后才执行 |
| 纠错 | "不对" 重新描述 | 支持撤销重来 |

## 4. 功能需求

### 4.1 语音交互模块（P0）

| ID | 功能 | 描述 | 优先级 |
|---|---|---|---|
| V-01 | 唤醒词检测 | 检测"小智"/"小zhi"同音词，触发监听模式 | P0 |
| V-02 | 语音采集 | getUserMedia 获取麦克风权限 | P0 |
| V-03 | ASR 转文字 | Web Speech API 中文语音识别 | P0 |
| V-04 | 识别文本展示 | 实时展示"你说的：xxx" | P0 |
| V-05 | 结束语检测 | 检测"开始吧"/"可以了"/"好的"/"行"等确认词 | P0 |
| V-06 | 纠错支持 | 检测"不对"/"不是"/"重新来"等纠错词 | P0 |

### 4.2 AI Agent 模块（P0）

| ID | 功能 | 描述 | 优先级 |
|---|---|---|---|
| A-01 | 意图分析 | LLM 分析用户绘图意图 | P0 |
| A-02 | 多轮确认对话 | LLM 反问确认需求细节 | P0 |
| A-03 | Function Calling | 确认后选择工具并填充参数 | P0 |
| A-04 | 画布上下文注入 | 当前元素列表摘要注入 LLM 上下文 | P0 |
| A-05 | 模糊指令消歧 | "大一点"→推断当前选中元素→scale(1.2) | P0 |
| A-06 | 错误恢复 | 工具调用失败时提示用户重试 | P0 |

### 4.3 绘图工具函数（P0，15 个）

**基础图形（6）**：

| ID | 工具 | 参数 | 描述 |
|---|---|---|---|
| T-01 | `drawRect` | position, size, style(fill,stroke,cornerRadius) | 画矩形 |
| T-02 | `drawCircle` | position(cx,cy), r, style | 画圆形 |
| T-03 | `drawEllipse` | position(cx,cy), rx, ry, style | 画椭圆 |
| T-04 | `drawLine` | start(x1,y1), end(x2,y2), stroke, strokeWidth | 画线段 |
| T-05 | `drawText` | position(x,y), text, fontSize, fontFamily, fill | 画文字 |
| T-06 | `drawTriangle` | points(x1,y1,x2,y2,x3,y3), style | 画三角形 |

**元素操作（6）**：

| ID | 工具 | 参数 | 描述 |
|---|---|---|---|
| T-07 | `selectElement` | id/description | 选中元素 |
| T-08 | `updateElement` | id, properties | 修改属性 |
| T-09 | `deleteElement` | id | 删除元素 |
| T-10 | `moveElement` | id, dx, dy / semantic("左边") | 移动元素 |
| T-11 | `scaleElement` | id, scale / semantic("大一点") | 缩放元素 |
| T-12 | `duplicateElement` | id, dx, dy | 复制元素 |

**画布操作（3）**：

| ID | 工具 | 参数 | 描述 |
|---|---|---|---|
| T-13 | `clearCanvas` | — | 清空画布 |
| T-14 | `undoAction` | — | 撤销 |
| T-15 | `exportImage` | format("svg"/"png"), filename | 导出 |

### 4.4 SVG 渲染模块（P0）

| ID | 功能 | 描述 |
|---|---|---|
| S-01 | React SVG 渲染 | 声明式 SVG 元素管理 |

### 4.5 导出模块（P0）

## 5. 非功能需求

| 类型 | 要求 |
|---|---|
| 性能 | 端到端延迟 P50 <3s（唤醒→确认→渲染） |

## 6. 用户场景

### 场景 1：首次使用画简单图形

```
用户打开页面 → 说"小智小智" → 系统提示"我在听"
→ "画一个蓝色正方形"
→ 系统: "你想画一个蓝色正方形，放在画布中间，对吗？"
→ "对"
→ 系统: "好的，开始吧？"
→ "可以了"
→ 画布出现蓝色正方形
```

### 场景 2：修改现有图形

```
(画布上已有一个蓝色正方形)
→ "小智小智" → "把那个正方形改成红色，大一点"
→ 系统: "你想把画布上的蓝色正方形改成红色，并放大一些，对吗？"
→ "对"
→ "开始吧"
→ 正方形变成红色并放大
```

### 场景 3：纠错

```
→ "小智小智" → "画一个三角形"
→ 系统: "你想画一个三角形，放在画布中间，对吗？"
→ "不对，画一个绿色的三角形在左上角"
→ 系统: "好的，一个绿色三角形，放在画布左上角。开始吧？"
→ "可以了"
→ 画布左上角出现绿色三角形
```
