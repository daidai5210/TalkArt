## 3. 最终确认需求

### 3.1 产品定位

纯语音控制的 AI SVG 矢量绘图 Web 工具。用户通过唤醒词激活，语音描述需求，AI Agent 多轮反问确认后，通过 Function Calling 调用绘图工具生成/操作 SVG 图形。

### 3.2 核心交互流程

```
唤醒词("小智小智") → 语音输入 → ASR 转文字
  → AI Agent 分析意图
  → 反问确认(多轮对话)
  → 用户说"开始吧"/"可以了"
  → AI Agent 调用工具函数
  → SVG 渲染
```

### 3.3 确认范围

| 维度 | 确认内容 |
|---|---|
| 产品形态 | Web 应用，浏览器原生 |
| 目标用户 | 大众创作者，自然覆盖无障碍场景 |
| 核心技术路线 | Voice → ASR → LLM Function Calling → 绘图工具 → SVG 渲染 |
| 唤醒词 | "小智"/"小zhi" 同音模糊匹配 |
| 确认机制 | AI Agent 多轮反问确认，确认后才执行 |
| 执行触发词 | "开始吧"、"可以了"等 |
| MVP 工具数量 | 15 个 P0 工具函数 |
| 不做项 | 登录、用户管理、数据库、协作、图层 |

### 3.4 验收标准

- 唤醒词可触发监听
- 语音→ASR→LLM 确认对话→工具执行→SVG 全链路可用
- 15 个 P0 工具函数正确执行
- 工具选择准确率 >90%
- 端到端延迟 P50 <3 秒
- 导出 SVG/PNG 正常
