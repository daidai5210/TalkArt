# 交互设计说明：TalkArt

> **版本**: v0.1.0-mvp

---

## 一、设计原则

| 原则 | 说明 |
|---|---|
| 纯语音优先 | 所有操作通过语音完成，UI 仅作辅助反馈 |
| 零键盘鼠标 | 用户不应需要任何物理输入设备 |
| 即时反馈 | 每个语音输入都有视觉反馈 |
| 容错设计 | 支持纠错、重说、文本兜底 |

## 三、交互状态

| 状态 | 视觉指示 | 用户可做什么 |
|---|---|---|
| **IDLE** | 🟢 "等待唤醒..." | 说唤醒词 |
| **WAKE_WORD** | 🟡 "我在听..." | 说绘图需求 |
| **LISTENING** | 🔴 录音动画 | 说话 |
| **CONFIRMING** | 💬 显示 AI 反问气泡 | 说"对"/"开始吧"/"不对" |
| **EXECUTING** | ⏳ 骨架动画 | 等待 |
| **ERROR** | ❌ 错误提示 | 重新说 |

## 四、语音交互规范

### 4.1 唤醒

- 唤醒词："小智小智" / "嘿小智" / "小智"
- 反馈：状态指示器变黄 + 提示音（可选）
- 超时：5 秒无语音输入回到 IDLE

### 4.2 确认对话

- AI 反问以气泡形式展示在画布上方
- 用户回答后气泡更新
- 确认完成（"开始吧"）后气泡消失，画布更新

### 4.3 错误处理

- ASR 识别为空：提示"没有听清，请再说一次"
- LLM 超时：提示"正在思考，请稍候..."
- 工具执行失败：提示具体错误 + 建议重试

## 五、无障碍设计

| 特性 | 说明 |
|---|---|
| 高对比度模式 | 支持系统偏好 |
| 大字体选项 | 识别文本和 AI 回复字体可调 |
| 屏幕阅读器 | ARIA 标签覆盖所有交互元素 |
| 键盘兜底 | 虽然目标是纯语音，但保留空格键激活麦克风 |
