## 二、研究背景

用户希望开发一款**纯语音控制的 AI SVG 矢量绘图 Web 工具**。用户不能使用鼠标或键盘，仅通过语音指令完成绘图创作。

核心差异化：不是 AI 文生图（Midjourney/DALL·E 路线），而是**指令绘图**——语音转文字后，由 LLM 通过 Function Calling 识别意图并调用预定义的绘图工具函数，在网页中生成/操作 SVG 图形。

经过 5 轮需求对话确认，最终方向如下：
- 产品形态：Web 应用（P1），浏览器原生，零安装
- 目标用户：所有人（U1）——不想学复杂绘图软件的大众创作者，自然覆盖无障碍场景
- AI 能力：指令控制型（C2 + Agent）——LLM Function Calling 调用绘图工具执行 SVG 操作

## 需求对话过程

### 第 1 轮 - 2026-06-11

**用户原始需求**：开发一款纯语音控制的绘图工具。用户不能使用鼠标或键盘，仅通过语音指令完成绘图创作。需要综合考虑指令理解的准确性与容错性、语音到绘图操作的响应延迟，以及复杂指令的拆解与执行能力。实现应用的同时，额外提交设计文档。

**主 AIC 质疑与澄清**：
- "纯语音控制"与"像素级操作"存在精度矛盾，需确认交互模式
- 指令歧义问题：模糊指令需要大量上下文推理
- 延迟体验要求：ASR → NLP → 执行需控制在 300-500ms
- 询问调研深度、目标用户、产品形态

**用户选择**：方案 A（全面调研）

### 第 2 轮 - 2026-06-11

**主 AIC 进一步澄清**：目标用户（U1-U4）、AI 能力边界（C1-C3）、产品形态（P1-P3）三个维度的选择

**用户选择**：P1（Web 应用）。其余维度默认推荐

### 第 3 轮 - 2026-06-11

**主 AIC 确认方向**：综合前面的选择做方向总结

**用户确认**：研究方向无误

### 第 4 轮 - 2026-06-11

**用户主动澄清**：「U1C1 就是我们设计一些绘图的 tools，然后语音转文字之后，由 AI 识别意图，采用 function call 的形式调用工具执行工具进行绘图」

**主 AIC 调整理解**：这是 Agentic Drawing Tool 路线——AI Agent 通过 Function Calling 操作绘图工具，不是 AI 文生图

### 第 5 轮 - 2026-06-11

**用户最终确认**：「不是文生图，而是指令绘图，绘制 SVG，就是在网页中绘制想要的图」

**主 AIC 确认**：SVG 矢量绘图，LLM Function Calling → 工具函数 → SVG 元素渲染
