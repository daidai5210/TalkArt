## 二、画像 1：大众创作者（主要画像）

### 2.1 画像描述

| 维度 | 描述 |
|---|---|
| **姓名** | 小李 |
| **年龄** | 28 |
| **职业** | 市场营销专员 |
| **技术能力** | 中等（会用 Office，不会 Photoshop/Figma） |
| **绘图需求** | 制作简单的示意图、流程图、社交媒体配图 |
| **当前痛点** | 想画一个简单图但打开 Figma 不知道从哪开始 |
| **使用场景** | 晚上在家，想做一张说明图放到 PPT 里 |

### 2.2 场景故事

> 小李想画一个简单的流程图说明新项目的工作流程。他打开 Figma 花了 10 分钟不知道矩形工具在哪。他想要的就是说一句"画三个并排的方框，用箭头连接"，然后看到图出来。

### 2.3 核心需求

| 需求 | 优先级 | 说明 |
|---|---|---|
| 零学习曲线 | P0 | 不需要知道"矩形工具"在哪，说就行 |
| 快速出图 | P0 | 30 秒内从想法到可视化 |
| 基础美观 | P1 | 图看起来不丑，有基本的美感 |
| 导出为 PNG/SVG | P1 | 能放到 PPT、文档里 |

## 三、画像 2：无障碍用户（自然覆盖画像）

### 3.1 画像描述

| 维度 | 描述 |
|---|---|
| **姓名** | 王老师 |
| **年龄** | 42 |
| **职业** | 中学物理教师 |
| **状况** | 上肢运动障碍（无法使用鼠标精细操作） |
| **技术能力** | 中等，会语音输入 |
| **绘图需求** | 制作物理示意图、力的分析图、电路简图 |
| **当前痛点** | 现有绘图工具都依赖鼠标，语音控制工具（Voice Access）只能模拟点击 |
| **使用场景** | 备课时需要快速画一张力的分解图 |

### 3.2 场景故事

> 王老师想画一张力的分解图。用 Voice Access 只能说"点击"、"拖拽"，画一条线需要 20 个语音指令。他需要的是一句话："画一个 30 度角的斜面，上面放一个方块，标出重力、支持力和摩擦力的方向"。

### 3.3 核心需求

| 需求 | 优先级 | 说明 |
|---|---|---|
| 语义级指令 | P0 | 不是"点击坐标 100,200"，而是"在这里画个圆" |
| 高容错率 | P0 | 指令模糊时能理解意图而非失败 |
| 逐元素可修改 | P1 | 画错了不需要全部重来 |
| WCAG 合规 | P2 | 长期考虑，MVP 不强制 |

## 六、用户画像优先级

| 画像 | MVP 优先级 | 理由 |
|---|---|---|
| **大众创作者（小李）** | **P0** | 最大用户基数，最直接的痛点，「不想学软件但想画图」 |
| **无障碍用户（王老师）** | **P0** | 最有意义的用户场景，差异化最强，社会价值高 |
| **快速原型（小张）** | P1 | 使用频率高但替代方案多 |
| **儿童教育（小明）** | P2 | 需要额外的安全机制和 UX 设计 |

## 七、用户旅程地图（语音绘图 vs 传统绘图）

### 7.1 传统绘图工具（如 Figma）

```
想画图 → 打开应用 → 找到工具 → 学习工具 → 开始拖拽 
→ 发现不对 → 撤销 → 重新拖拽 → 调整位置 → 调整颜色 
→ 导出 → 完成
⏱️ 总耗时：5-30 分钟（新手 30+ 分钟）
```

### 7.2 语音 AI 绘图工具（我们的目标）

```
想画图 → 打开网页 → 说话"画三个并排的蓝色方框" 
→ 看到结果 → "第二个高一点，第三个改成红色" 
→ 导出 → 完成
⏱️ 总耗时：30 秒-2 分钟
```

**效率提升：10-50x**（对非专业用户）
