## 一、可测试性评估

### 1.1 各功能模块可测试性

| 功能模块 | 可测试性 | 测试方式 | 自动化难度 |
|---|---|---|---|
| 语音采集（麦克风） | 🟢 高 | 浏览器模拟 + 真实设备 | 中 |
| ASR 转文字 | 🟢 高 | Mock ASR 输出 | 低 |
| LLM Function Calling | 🟡 中 | Mock LLM 响应 + E2E | 中 |
| 工具函数执行 | 🟢 高 | 单元测试 | 低 |
| SVG 渲染 | 🟢 高 | 单元测试 + 视觉快照 | 低 |
| 状态管理（撤销/重做） | 🟢 高 | 单元测试 | 低 |
| 导出 | 🟢 高 | 单元测试 | 低 |
| 端到端延迟 | 🟡 中 | 手动测试 + 性能打点 | 高 |
| 空间推理准确性 | 🟡 中 | 手动测试集 | 高 |

### 1.2 可测试性风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| LLM 输出非确定性 | 同一指令可能产生不同工具调用 | 宽松匹配断言 + 结果验证而非过程验证 |
| 语音识别环境差异 | 不同麦克风/噪音环境影响 ASR | 多环境手动测试 + 文本兜底 |
| 浏览器兼容性 | Web Speech API 在不同浏览器行为不同 | 多浏览器 E2E |

## 二、测试策略

### 2.1 测试金字塔

```
           ┌──────┐
           │ E2E  │  ~5%: 关键用户流程
           ├──────┤
           │ 集成  │  ~25%: LLM→工具函数链路
           ├──────┤
           │ 单元  │  ~70%: 工具函数、状态管理、SVG
           └──────┘
```

### 2.2 单元测试清单

| 测试对象 | 测试内容 | 优先级 |
|---|---|---|
| drawRect / drawCircle / ... | 参数正确→SVG 元素属性正确 | P0 |
| updateElement | 修改属性→SVG 元素更新 | P0 |
| deleteElement | 删除→元素从状态中移除 | P0 |
| moveElement / scaleElement | 变换正确 | P0 |
| undoAction / redoAction | 历史栈正确 | P0 |
| clearCanvas | 状态清空 | P0 |
| exportImage (SVG) | 输出合法 SVG 字符串 | P0 |
| exportImage (PNG) | 输出 PNG Blob | P0 |
| 画布上下文摘要生成 | 摘要格式正确、不溢出 | P0 |
| 语义坐标计算 | "center"→正确坐标 | P0 |

### 2.3 集成测试清单

| 测试场景 | 测试步骤 | 验证点 |
|---|---|---|
| 语音→ASR→LLM→工具→SVG 全链路 | Mock ASR "画一个红色圆"→LLM→drawCircle→SVG | 画布出现红色圆 |
| 增量修改 | Mock ASR "把那个改成蓝色"→LLM→updateElement | 圆的颜色变成蓝色 |
| 多步操作 | Mock ASR 连续 3 条指令 | 画布状态正确累积 |
| 撤销/重做 | 画→撤销→重做 | 画布状态回退和恢复 |
| LLM 错误恢复 | Mock LLM 返回无效函数调用 | 系统不崩溃，提示用户 |

### 2.4 E2E 测试清单

| 测试场景 | 优先级 | 说明 |
|---|---|---|
| 打开应用→画图→导出 SVG | P0 | 核心流程可用 |
| 打开应用→画图→修改→撤销→导出 PNG | P0 | 核心编辑流程 |
| 文本兜底输入 | P1 | 语音失败时的备用方案 |
| 多浏览器兼容性 | P1 | Chrome / Firefox / Edge / Safari |

## 三、验证指标

### 3.1 功能正确性指标

| 指标 | 目标 | 测量方式 |
|---|---|---|
| 工具函数正确率 | 100% | 单元测试通过率 |
| Function Calling 工具选择准确率 | >90% | 手动测试：20 个标准指令，LLM 选择正确的工具 |
| 参数填充准确率 | >85% | 手动测试：参数在合理范围内 |
| 端到端指令成功率 | >80% | 50 个指令的 E2E 测试集 |

### 3.2 性能指标

| 指标 | MVP 目标 | 生产目标 | 测量方式 |
|---|---|---|---|
| ASR 延迟 | <200ms | <100ms | Performance API |
| LLM 响应延迟（P50） | <2s | <1s | API 耗时打点 |
| LLM 响应延迟（P95） | <5s | <3s | API 耗时打点 |
| 端到端延迟（P50） | <3s | <2s | 从"松手"到"SVG 渲染" |
| 工具执行延迟 | <10ms | <5ms | 函数耗时打点 |
| SVG 渲染延迟 | <16ms | <16ms | 60fps 保证 |
| 画布操作 FPS | >55fps | >58fps | requestAnimationFrame |
| 首屏加载 | <2s | <1s | Lighthouse |

### 3.3 用户体验指标

| 指标 | MVP 目标 | 测量方式 |
|---|---|---|
| 用户首次任务完成率 | >70% | 用户测试：给他一个任务"画三个排成一行的蓝色方块" |
| 用户满意度评分 | >3.5/5 | 测试后问卷 |
| 指令必说次数 | <1.5x（平均每指令重复 <0.5 次） | 测试记录 |

## 五、MVP 验证计划

### 5.1 验证目标

验证核心假设：**"用户可以通过语音指令成功创建和编辑矢量图形，且体验优于手动操作（对非专业用户）。"**

### 5.2 验证步骤

| 步骤 | 内容 | 成功标准 |
|---|---|---|
| 1. 内部功能测试 | 用 20 个标准指令测试全链路 | 工具选择准确率 >90%，参数合理率 >85% |
| 2. 延迟优化 | 测量并优化端到端延迟 | P50 <3s |
| 3. 用户测试（3-5 人） | 给用户 3 个绘图任务 | >70% 任务完成 |
| 4. 迭代修复 | 基于用户反馈修复关键问题 | — |
| 5. 验证结论 | 撰写 `mvp-validation-report.md` | — |

### 5.3 MVP 测试任务

| 任务 | 指令示例 | 验证点 |
|---|---|---|
| 任务 1：画基础图形 | "在画布中间画一个蓝色的圆" | 基础工具调用 |
| 任务 2：修改图形 | "把这个圆改成红色，大一点" | 增量修改 |
| 任务 3：组合图形 | "画一个黄色矩形，里面写'Hello'文字" | 多元素组合 |

## 六、已知测试盲区

| 盲区 | 风险 | MVP 是否覆盖 |
|---|---|---|
| 不同口音/语速 | ASR 识别率波动 | 否（MVP 后补测） |
| 噪音环境 | ASR 降噪能力 | 否 |
| 复杂指令拆解 | LLM 拆解失败 | 否（MVP 只测单步） |
| 大画布性能 | 100+元素性能 | 否（MVP 后压测） |
| 移动端浏览器 | 触屏 + 语音 | 否（先桌面端） |
