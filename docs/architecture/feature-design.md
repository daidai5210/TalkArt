├── 语音交互

│   ├── 多轮反问确认

│   ├── 画布上下文注入
│   ├── 模糊指令消歧
│   └── 错误恢复
├── 绘图工具
│   ├── 基础图形 ×6
│   ├── 元素操作 ×6
│   └── 画布操作 ×3
├── SVG 渲染
│   ├── React SVG 画布
│   ├── 元素选中高亮
│   ├── 实时预览
│   └── 状态管理 (Zustand)

## 二、交互状态机

```
                    ┌─────────┐
                    │  IDLE   │ ← 初始状态
                    └────┬────┘
                         │ 检测到"小zhi"
                         ▼
                    ┌─────────┐
                    │WAKE_WORD│ 唤醒
                    └────┬────┘
                         │ 开始录音
                         ▼
                    ┌─────────┐
                    │LISTENING│ 监听中
                    └────┬────┘
                         │ ASR 完成
                         ▼
                    ┌─────────┐
                    │CONFIRM  │ AI 反问确认
                    │ _ING    │
                    └────┬────┘
                         │
              ┌──────────┼──────────┐
              │ "不对"    │ "开始吧"   │
              ▼          ▼          │
         LISTENING    ┌─────────┐   │
         (重新描述)    │EXECUTING│   │
                      └────┬────┘   │
                           │ 工具执行│
                           ▼        │
                      ┌─────────┐   │
                      │  IDLE   │←──┘
                      └─────────┘
```

## 三、关键交互设计

### 3.1 唤醒词检测

- 关键词："小智" / "小zhi" / "小志" / "小知"（同音即可）
- 触发位置：任何语音输入的开头或中间
- 容错：宽松匹配，不需要精确发音
- 实现：在 ASR 连续结果中正则匹配 `xiao\s*zhi`

### 3.2 多轮确认策略

| 指令类型 | 确认轮数 | 示例 |
|---|---|---|
| 简单明确 | 1 轮 | "画红色圆形"→"好的，画红色圆形在中间。开始吧？" |
| 有歧义 | 2 轮 | "画个图"→"什么类型？"→"流程图"→确认→执行 |
| 含修改 | 内联确认 | "大一点"→推断当前选中元素→"把蓝色矩形放大到 120% 对吗？" |

### 3.3 结束语检测

- 触发词："开始吧" / "可以了" / "好的" / "行" / "画吧" / "执行"
- 逻辑：只在 CONFIRMING 状态下检测
- 行为：检测到后立即调用 LLM 做最终 Function Calling

### 3.4 纠错流程

```
用户: "不对" / "不是" / "重新来" / "换个"
  ↓
系统清除当前确认状态
  ↓
返回 LISTENING，等待重新描述
```
